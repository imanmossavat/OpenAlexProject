# Structure & Config Map

Use this sheet to quickly locate critical code, configuration, operational files, and documentation references.

## Environments & Secrets

| Purpose | Location | Notes |
| ------- | -------- | ----- |
| Global env template | `.env`, `.env.docker.example` (repo root) | Created by `install.py`; copy Docker example when containerizing. Documented in `Installation guide.md`. |
| Backend env | `article-crawler-backend/.env` | Generated by installer; consumed by FastAPI. Includes `ARTICLECRAWLER_PATH`, `STORAGE_ROOT`, `OPENALEX_EMAIL`, optional `ZOTERO_*`, `GROBID_URL`. |
| Frontend env | `frontend/.env` | Vite variables (`VITE_API_URL`, analytics toggles if added). |
| Crawler env | `fakenewscitationnetwork/.env` | API keys, Zotero credentials, optional job tuning for CLI runs. |
| Docker env | Root `.env` consumed by `docker-compose.yml` | Copy from `.env.docker.example` and set host paths/ports for containers. |

| -------- | -------- | ----- |

## Diagnostics & Logs

| Signal | Location | Notes |
| ------ | -------- | ----- |
| Crawler logs | `StorageAndLoggingConfig.log_folder` (per experiment) | Paths defined in `StorageAndLoggingConfig.folders_all`; inspect when crawls fail/resume. |
| Vault manifests | `vault/run.md`, `vault/README.md` inside each experiment | Summaries of config, seeds, errorsâ€”useful for debugging and sharing context with stakeholders. |
| Backend logs | Uvicorn / `ArticleCrawlerAPI` logger | Contains job_id-scoped lines when crawler jobs start/resume/save. |
| Job metadata | `app/core/stores/crawler_job_store.py` (default in-memory store) | Inspect `status`, `iterations_completed`, `error_message`, `config_snapshot`. Replace with persistent store if needed. |
| Frontend logs | Browser console + Vite dev server output | Shows API failures from `shared/api/client.js` or workflow errors. |
| Retraction version | `article-crawler-backend/retraction_cache/retraction_watch_version.txt` | Confirms CSV freshness; mismatch explains missing flags. |
